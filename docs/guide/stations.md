## คิวงานในสถานี (บล็อกคิว)

ถ้าแบบสถานีมีบล็อก **คิวงานปัจจุบัน (StationQueueBlock)** มักจะแบ่งเป็น 2 โซน:

- **งานด่วน** — ออเดอร์ที่มีความเร่งด่วนสูง (เช่นใกล้กำหนดส่งระดับวิกฤต) เพื่อให้เห็นก่อน
- **คิวงานถัดไป** — ออเดอร์และชิ้นงานอื่น ๆ ที่อยู่ที่สถานีนี้ รวมถึงที่กำลังเริ่มทำหรือทำเสร็จแล้วแต่ยังอยู่ในคิวของสถานี (ไม่ถูกดันขึ้นไปบนสุดเพียงเพราะกดเริ่ม/เสร็จ)

นอกจากนี้ยังมีบล็อก **ประวัติการผลิต (StationHistory)** สำหรับชิ้นที่ผ่านสถานีแล้วหรือรอสแกนออก — ขึ้นกับที่ผู้ดูแลออกแบบหน้า

::: tip อัปเดตอัตโนมัติ
ข้อมูลที่หน้าสถานีจะอัปเดตแบบ **เรียลไทม์** — ไม่ต้องกดรีเฟรชหน้าจอ เมื่อมีชิ้นงานใหม่เข้ามาหรือสถานะเปลี่ยน จะเห็นทันที
:::

## เบิกและเคลมจากหน้าสถานี

ที่แถบด้านบนของหน้าสถานีมักมีปุ่ม:

- **เบิก** — เปิดหน้าต่างเบิกกระจกตามแผ่น: สแกนหรือพิมพ์เลขกระจก เลือกสต็อก ยืนยันเบิก  
  - เมื่อสำเร็จ ระบบแสดง **เลขที่ใบเบิก** แบบอ่านง่ายจาก API (รูปแบบ `WDW-xxxx`) ไม่ใช่แค่รหัสฐานข้อมูลยาว ๆ  
- **เคลม** — เปิดหน้าต่างแจ้งเคลมตามแผ่น: พิมพ์รหัสกระจกแล้วกด **Enter** (หรือกดปุ่มข้างช่อง) เพื่อโหลดข้อมูลแผ่น กรอกเหตุผล แนบรูป แล้วส่งเรื่อง

## หน้าสแกนสำหรับมือถือ (`/stations/.../scan`)

ใช้เมื่อต้องการสแกนยิงบาร์โค้ด/QR แบบรวดเร็ว หลังกด **เสร็จสิ้น** ระบบจะแสดงข้อความว่าส่งต่อไปสถานีใด — ข้อมูลนี้มาจาก **API** (`nextStation`) ไม่ใช่การคำนวณฝั่งหน้าเว็บเอง

บางสถานี (เช่น ตัด) อาจบังคับให้ **เบิกวัสดุก่อน** จึงจะกดเสร็จสิ้นได้ — ขึ้นกับการตั้งค่าในแบบสถานี

## สถานี QC (ตรวจคุณภาพ)

ถ้าแบบสถานีมีบล็อก **QC Inspector** พนักงานจะติ๊กรายการตรวจแล้วกด **ผ่าน QC** หรือ **ไม่ผ่าน**  
ระบบจะส่งคำสั่งไปยัง API ตามลำดับที่เซิร์ฟเวอร์กำหนด (รวมถึงขั้น **เริ่มงาน / เสร็จสิ้น** ที่สถานี QC ก่อนบันทึกผลผ่านหรือไม่ผ่าน) — หากข้ามขั้นนี้ API จะแจ้งว่าต้องกดเสร็จสิ้นก่อน
