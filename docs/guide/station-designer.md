# ออกแบบสถานี

<Card title="ออกแบบสถานีคืออะไร?" variant="blue">
เครื่องมือสร้างหน้าจอสำหรับแต่ละสถานีงานในโรงงาน — คุณสามารถลาก-วางบล็อกต่าง ๆ เพื่อสร้างหน้าจอที่เหมาะกับงานของสถานีนั้น ๆ ได้เองโดยไม่ต้องเขียนโปรแกรม
</Card>

::: warning สิทธิ์การเข้าถึง
หน้าออกแบบสถานีเข้าถึงได้เฉพาะ **Admin** เท่านั้น
:::

## ภาพรวม

ระบบแบ่งออกเป็น 2 ส่วนหลัก:

- **คลังแบบสถานี** — แสดงรายการแบบสถานีที่สร้างไว้ทั้งหมด
- **ตัวแก้ไขแบบสถานี** — หน้าออกแบบแบบลาก-วาง

## วิธีสร้างแบบสถานีใหม่

## หน้าตัวแก้ไข

หน้าออกแบบแบ่งเป็น 3 ส่วน:

| ส่วน | รายละเอียด |
|------|-----------|
| **แผงบล็อก (ซ้าย)** | รายการบล็อกทั้งหมดที่ใช้ได้ คลิกหรือลากมาวางในพื้นที่ออกแบบ |
| **พื้นที่ออกแบบ (กลาง)** | พื้นที่แสดงตัวอย่างหน้าจอ ลาก ย้าย จัดเรียงบล็อกได้ตามต้องการ |
| **แผงตั้งค่า (ขวา)** | ค่าต่าง ๆ ของบล็อกที่คลิกเลือกอยู่ เช่น ข้อความ สี ขนาด ฯลฯ |

::: tip คีย์ลัด
- **Ctrl+Z** (Windows) / **Cmd+Z** (Mac) — เลิกทำ
- **Ctrl+Y** (Windows) / **Cmd+Shift+Z** (Mac) — ทำซ้ำ
:::

## บล็อกทั้งหมด

บล็อกแบ่งออกเป็น 5 หมวด ดังนี้

---

### 🔲 บล็อกโครงสร้าง

บล็อกกลุ่มนี้ใช้จัดวางโครงร่างหน้าจอ ก่อนเพิ่มเนื้อหา

---

#### กล่องพื้นที่ (Section)

กล่องขนาดใหญ่ที่ใช้จัดกลุ่มบล็อกอื่น ๆ เข้าด้วยกัน เช่น จัดกลุ่มส่วน "ข้อมูลออเดอร์" แยกจากส่วน "รายการชิ้นงาน" ตั้งสีพื้นหลังเพื่อให้แยกส่วนได้ชัดเจน

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| สีพื้นหลัง | ขาว / เทา / น้ำเงินอ่อน / เขียวอ่อน / เหลืองอ่อน |
| ระยะห่างรอบใน | ไม่มี / เล็ก / กลาง / ใหญ่ |

---

#### วางเคียงกัน (TwoColumns)

แบ่งพื้นที่ออกเป็น 2–4 ช่องแนวตั้งเคียงกัน เหมาะเมื่อต้องการวางบล็อกสองอย่างไว้ข้างกัน เช่น "รายการออเดอร์" อยู่ซ้าย และ "รายละเอียดชิ้นงาน" อยู่ขวา

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| จำนวนช่อง | 2 / 3 / 4 |
| อัตราส่วนความกว้าง | เท่ากัน / ซ้ายกว้างกว่า (2:1) / ขวากว้างกว่า (1:2) / ซ้ายกว้างมาก (3:1) / ขวากว้างมาก (1:3) |
| ช่องว่างระหว่างคอลัมน์ | เล็ก / กลาง / ใหญ่ / ใหญ่มาก |

---

#### คอลัมน์ (Column)

กล่องเรียงแนวตั้ง ใช้วางบล็อกซ้อนกันภายในช่อง "วางเคียงกัน" บล็อกนี้จะสร้างขึ้นอัตโนมัติเมื่อเพิ่ม "วางเคียงกัน"

---

#### เส้นคั่น (Divider)

เส้นแนวนอนสำหรับแบ่งส่วนเนื้อหาให้ดูเป็นระเบียบ

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| ระยะห่างด้านบน-ล่าง | เล็ก / กลาง / ใหญ่ |
| รูปแบบเส้น | ทึบ / ประ |

---

#### พื้นที่ว่าง (Spacer)

ช่องว่างแนวตั้ง ใช้เพิ่มระยะห่างระหว่างบล็อก กำหนดความสูงเป็นตัวเลข เช่น 16 หรือ 32

---

### 📝 บล็อกข้อความและแสดงผล

บล็อกกลุ่มนี้ใช้แสดงข้อความและข้อมูล

---

#### หัวเรื่อง (Heading)

แสดงข้อความหัวเรื่องขนาดใหญ่ ใช้ตั้งชื่อส่วนต่าง ๆ ในหน้าจอ นอกจากข้อความตายตัว ยังสามารถแสดงข้อมูลจากออเดอร์หรือบิลที่เลือกอยู่ได้โดยอัตโนมัติ

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| ข้อความ | พิมพ์ข้อความที่ต้องการแสดง |
| ขนาด | ใหญ่สุด / ใหญ่ / กลาง / เล็ก |
| การจัดตำแหน่ง | ซ้าย / กลาง / ขวา |
| สี | ปกติ / เน้น / จาง / น้ำเงิน / เขียว |
| รูปแบบ | ปกติ / หนา / เอียง / หนา+เอียง |
| แสดงข้อมูลอัตโนมัติ | เลือกข้อมูลที่ต้องการดึงมาแสดง เช่น ชื่อลูกค้า, รหัสออเดอร์ (ดูตารางข้อมูลที่แสดงได้ด้านล่าง) |

---

#### ข้อความ (Paragraph)

แสดงข้อความเนื้อหาทั่วไป เหมาะสำหรับคำอธิบาย หมายเหตุ หรือข้อความแนะนำขั้นตอนให้พนักงาน

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| ข้อความ | พิมพ์ข้อความที่ต้องการ |
| การจัดตำแหน่ง | ซ้าย / กลาง / ขวา |
| ขนาด | เล็ก / ปกติ / ใหญ่ |
| แสดงข้อมูลอัตโนมัติ | เลือกข้อมูลที่ต้องการดึงมาแสดงแทนข้อความ |

---

#### ป้ายสถานะ (Badge)

แสดงป้ายข้อความสั้น ๆ รูปแบบมน ๆ เหมาะใช้แสดงสถานะหรือข้อมูลเน้น เช่น "รออนุมัติ" หรือ "เสร็จแล้ว"

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| ข้อความ | ข้อความบนป้าย |
| สี | ปกติ (เทา) / เขียว / เหลือง / แดง / น้ำเงิน |
| แสดงข้อมูลอัตโนมัติ | เลือกข้อมูลสถานะจากออเดอร์หรือชิ้นงาน |

---

#### การ์ดข้อมูล (InfoCard)

การ์ดแสดงข้อมูลสำคัญ มีเส้นสีด้านซ้ายเพื่อเน้น เหมาะสำหรับแสดงข้อมูลที่พนักงานต้องสังเกตก่อนทำงาน เช่น ข้อความเตือน หรือรายละเอียดพิเศษของออเดอร์

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| หัวข้อ | ข้อความบรรทัดแรก (ตัวหนา) |
| หัวข้อย่อย | ข้อความบรรทัดที่สอง (ตัวเล็กกว่า) |
| เนื้อหา | รายละเอียดหลัก |
| สีเส้นเน้น | น้ำเงิน / เขียว / ส้ม / ม่วง / แดง / เทา |

---

#### ตัวบ่งชี้สถานะ (StatusIndicator)

แสดงสถานะในรูปแบบต่าง ๆ เหมาะสำหรับแสดงสถานะของชิ้นงานหรือออเดอร์ที่เลือกอยู่

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| ชื่อป้ายกำกับ | ข้อความที่แสดงคู่กับสถานะ |
| รูปแบบ | แบบกลม / แบบป้าย / แบบจุด / แบบแท็ก |
| แสดงข้อมูลอัตโนมัติ | เลือกข้อมูลสถานะ |

---

### 🖊️ บล็อกรับข้อมูล

บล็อกกลุ่มนี้ใช้รับข้อมูลจากผู้ใช้ เช่น การกรอก การเลือก และการสแกน

---

#### ช่องกรอกข้อมูล (InputField)

ช่องให้พนักงานพิมพ์ข้อมูล เช่น จำนวน หมายเหตุ หรือวันที่

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ชื่อป้ายกำกับ | ข้อความที่แสดงเหนือช่องกรอก เช่น "จำนวน" |
| ข้อความตัวอย่าง | ข้อความแสดงจางๆ ก่อนกรอก เช่น "กรอกจำนวนชิ้นงาน" |
| ประเภท | ข้อความ / ตัวเลข / วันที่ / อีเมล / เบอร์โทร |
| บังคับกรอก | เปิด = ห้ามบันทึกถ้ายังไม่กรอก |
| ชื่ออ้างอิง | ชื่อสำหรับระบบใช้จดจำข้อมูลในช่องนี้ เพื่อให้ปุ่มหรือบล็อกอื่นนำไปใช้ต่อได้ ตั้งชื่อสั้น ๆ ภาษาอังกฤษ เช่น `note` หรือ `quantity` |
| ค่าเริ่มต้น | ข้อมูลที่ระบบกรอกให้อัตโนมัติก่อนที่พนักงานจะแก้ไข |

---

#### ช่องเลือกจากรายการ (SelectField)

Dropdown ให้พนักงานเลือกข้อมูลจากรายการ ระบบจะโหลดรายการล่าสุดจากข้อมูลในระบบให้อัตโนมัติ เมื่อเลือกรายการใด ข้อมูลทั้งหมดของรายการนั้นจะถูกนำมาใช้ในบล็อกอื่น ๆ ในหน้านี้ได้ทันที

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ชื่อป้ายกำกับ | ข้อความที่แสดงเหนือ Dropdown เช่น "เลือกบิล" |
| ชนิดรายการ | ดูตารางด้านล่าง |
| ชื่ออ้างอิง | ชื่อสำหรับระบบใช้จดจำว่าเลือกอะไร |
| กรองจากช่องอื่น | เชื่อมกับช่องเลือกอื่นในหน้านี้เพื่อกรองรายการ เช่น เลือกลูกค้าก่อน แล้วรายการออเดอร์จะแสดงเฉพาะของลูกค้านั้น |

**ชนิดรายการที่เลือกได้:**

| ชนิดรายการ | แสดงอะไร |
|-----------|---------|
| ออเดอร์ | รายการออเดอร์ทั้งหมดในระบบ |
| บิล | รายการบิลที่ยังไม่ได้สร้างออเดอร์ |
| วัสดุ | รายการวัสดุ/กระจกในคลัง |
| พนักงาน | รายชื่อพนักงานทั้งหมด |
| ลูกค้า | รายชื่อลูกค้าทั้งหมด |

::: tip เลือกบิลแล้วข้อมูลเข้าอัตโนมัติ
เมื่อพนักงานเลือกบิลจาก Dropdown ระบบจะดึงข้อมูลบิลนั้นมาทั้งหมด — ชื่อลูกค้า จำนวน รายละเอียด — เพื่อให้บล็อกอื่นในหน้านี้แสดงผลและปุ่มนำไปสร้างออเดอร์ได้ทันที
:::

---

#### ช่องกรอกข้อความยาว (TextAreaField)

ช่องให้พนักงานพิมพ์ข้อความหลายบรรทัด เหมาะสำหรับบันทึก คำอธิบายงาน หรือหมายเหตุพิเศษ

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ชื่อป้ายกำกับ | ข้อความที่แสดงเหนือช่อง |
| ข้อความตัวอย่าง | ข้อความแสดงจางๆ ก่อนกรอก |
| ความสูงช่อง | จำนวนแถว เช่น 3 = สูงพอกรอก 3 บรรทัด |
| ชื่ออ้างอิง | ชื่อสำหรับระบบใช้จดจำข้อมูล |

---

#### ปุ่มกด (ButtonBlock)

ปุ่มที่พนักงานกดเพื่อดำเนินการต่าง ๆ เช่น บันทึกข้อมูล สร้างออเดอร์ หรือไปยังหน้าอื่น

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| ข้อความบนปุ่ม | ข้อความที่แสดงบนปุ่ม เช่น "ยืนยันออเดอร์" |
| รูปแบบ | น้ำเงิน (หลัก) / ขอบ (ไม่เติม) / แดง (ลบ) / เขียว (สำเร็จ) |
| ขนาด | เล็ก / กลาง / ใหญ่ |
| เต็มความกว้าง | เปิด = ปุ่มขยายเต็มแนวนอน |
| การจัดตำแหน่ง | ซ้าย / กลาง / ขวา |

**สิ่งที่ปุ่มทำได้เมื่อกด:**

| ประเภทการทำงาน | รายละเอียด |
|--------------|-----------|
| **บันทึกข้อมูลในออเดอร์** | ตรวจสอบข้อมูลที่กรอกทั้งหมด แล้วบันทึกลงในออเดอร์ปัจจุบัน ใช้สำหรับอัปเดตสถานะหรือเพิ่มข้อมูลในออเดอร์ที่เปิดอยู่ |
| **ไปยังหน้าอื่น** | พาผู้ใช้ไปยังหน้าที่กำหนด เช่น กลับหน้ารายการสถานี หรือไปหน้าติดตามการผลิต กำหนดได้ว่าจะไปหน้าไหน |
| **ดำเนินการกับข้อมูล** | สั่งให้ระบบทำงาน เช่น สร้างออเดอร์ใหม่จากบิลที่เลือก หรืออัปเดตสถานะชิ้นงาน ระบบจะนำข้อมูลที่กรอกและเลือกไว้ทั้งหมดส่งเข้าระบบโดยอัตโนมัติ |
| **ยืนยันก่อนทำ** | แสดงกล่องป๊อปอัปให้ยืนยันก่อน แล้วจึงดำเนินการ ป้องกันการกดปุ่มโดยไม่ตั้งใจ |

::: tip ปุ่มสร้างออเดอร์
ตั้งประเภทการทำงานเป็น **ดำเนินการกับข้อมูล** และเลือกการทำงาน "สร้างออเดอร์ใหม่" ระบบจะนำข้อมูลจากบิลที่เลือกและช่องกรอกต่าง ๆ ไปสร้างออเดอร์ พร้อมโยงชิ้นงานเข้าออเดอร์ให้อัตโนมัติ
:::

#### สแกน QR Code (QrScanBlock)

ช่องให้พนักงานสแกน QR Code จากกล้องหรือเครื่องสแกน เพื่อโหลดข้อมูลหรือบันทึกการทำงาน

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ชื่อป้ายกำกับ | ข้อความแสดงเหนือช่องสแกน เช่น "สแกน QR ชิ้นงาน" |
| โหมดสแกน | ดูตารางด้านล่าง |
| บันทึกอัตโนมัติหลังสแกน | เปิด = เมื่อสแกนสำเร็จ ระบบจะบันทึกการเปลี่ยนแปลงที่กำหนดไว้ให้ทันที โดยไม่ต้องกดปุ่มเพิ่ม |
| ข้อมูลที่จะบันทึก | ข้อมูลที่ระบบบันทึกอัตโนมัติ เช่น สถานะ "เริ่มแล้ว" — กำหนดโดย Admin |
| ข้อความเมื่อสำเร็จ | ข้อความที่แสดงให้พนักงานเห็นหลังสแกนสำเร็จ |
| เปิดกล้อง | เปิด = มีปุ่มเปิดกล้องโทรศัพท์/กล้องเว็บแคมเพื่อสแกน |

**โหมดสแกน:**

| โหมด | สิ่งที่เกิดขึ้นเมื่อสแกน |
|------|------------------------|
| **โหลดออเดอร์** | ดึงข้อมูลออเดอร์จาก QR มาแสดงในหน้าจอ เพื่อให้บล็อกอื่นแสดงรายละเอียดออเดอร์นั้น |
| **โหลดบิล** | ดึงข้อมูลบิลจาก QR มาแสดง |
| **โหลดชิ้นงาน** | ดึงข้อมูลชิ้นงานจาก QR มาแสดง ขนาด ประเภท สถานะ |
| **สแกนเข้าสถานี** | บันทึกว่าชิ้นงานนี้มาถึงสถานีนี้แล้ว แสดงปุ่มให้พนักงานกด "เริ่มงาน" ต่อ |
| **สแกนออกสถานี** | บันทึกว่าชิ้นงานนี้ทำเสร็จและออกจากสถานีนี้แล้ว ระบบจะส่งชิ้นงานไปยังสถานีถัดไปโดยอัตโนมัติ |

บล็อกกลุ่มนี้ดึงข้อมูลจากระบบมาแสดงบนหน้าจอ และอัปเดตแบบเรียลไทม์

#### ตารางรายการ (RecordList)

แสดงรายการข้อมูลในรูปแบบตาราง ข้อมูลอัปเดตอัตโนมัติเมื่อมีการเปลี่ยนแปลง ใช้สำหรับแสดงรายการออเดอร์ที่รอ รายการบิล หรือรายการวัสดุในคลัง

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ชื่อป้ายกำกับ | ชื่อที่แสดงเป็นหัวตาราง |
| ชนิดข้อมูล | ออเดอร์ / บิล / วัสดุ / ชิ้นงาน / พนักงาน (เลือกชนิดเดียว) |
| แสดงสูงสุด | จำนวนแถวที่แสดง เช่น 5 หรือ 10 |
| ช่องค้นหา | เปิด = แสดงช่องค้นหาเหนือตาราง |
| กรองเฉพาะสถานีนี้ | เปิด = แสดงเฉพาะชิ้นงานที่อยู่ที่สถานีนี้ |
| แสดงปุ่ม QR | เปิด = ทุกแถวมีปุ่มเปิด QR Code |
| แสดงลิงก์ใบงาน | เปิด = ทุกแถวมีลิงก์เปิดใบงานเพื่อพิมพ์ |
| ซ่อนบิลที่เสร็จแล้ว | เปิด = ซ่อนบิลที่สร้างออเดอร์ไปแล้ว แสดงเฉพาะบิลที่ยังรอ |
| เฉพาะออเดอร์ที่มีงานค้าง | เปิด = แสดงเฉพาะออเดอร์ที่ยังมีชิ้นงานรอดำเนินการ |

**คอลัมน์ที่แสดงในตาราง:**

กำหนดได้ว่าตารางจะแสดงคอลัมน์อะไรบ้าง แต่ละคอลัมน์ต้องระบุ 3 อย่าง:

- **ชื่อข้อมูล** — ชื่อข้อมูลในระบบ (ดูตารางด้านล่าง)
- **ชื่อหัวคอลัมน์** — ข้อความที่แสดงบนหัวตาราง เช่น "ลูกค้า" หรือ "วันที่"
- **ประเภทการแสดงผล** — วิธีแสดงข้อมูล: ข้อความ / ตัวเลข / สถานะ / วันที่ / ป้าย / ราคา

ตัวอย่างคอลัมน์ที่ใช้บ่อย:

| ชื่อข้อมูล | ความหมาย | ประเภทที่แนะนำ |
|-----------|---------|--------------|
| `code` | รหัสออเดอร์/บิล | ข้อความ |
| `customer.name` | ชื่อลูกค้า | ข้อความ |
| `status` | สถานะ | สถานะ |
| `createdAt` | วันที่สร้าง | วันที่ |
| `quantity` | จำนวน | ตัวเลข |
| `totalPrice` | ราคารวม | ราคา |
| `material.name` | ชื่อวัสดุ | ข้อความ |

::: tip เมื่อคลิกแถวในตาราง
ข้อมูลแถวที่คลิกจะถูกโหลดเข้าระบบ — บล็อก "รายละเอียด" และ "หัวเรื่อง" ที่ตั้งค่าให้แสดงข้อมูลอัตโนมัติจะอัปเดตทันที สามารถกำหนดให้คลิกแล้วพาไปยังหน้าอื่นก็ได้
:::

#### รายละเอียดข้อมูล (RecordDetail)

แสดงรายละเอียดข้อมูลแบบแถวคู่ (ชื่อ: ค่า) เหมาะสำหรับแสดงข้อมูลครบถ้วนของออเดอร์หรือบิลที่เลือกอยู่ในปัจจุบัน

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ชื่อป้ายกำกับ | หัวข้อของส่วนรายละเอียด |
| ชนิดข้อมูล | ออเดอร์ / บิล / ชิ้นงาน ฯลฯ |

**ฟิลด์ที่แสดง:**

เลือกได้ว่าจะแสดงข้อมูลอะไรบ้าง แต่ละฟิลด์ระบุ:
- **ชื่อข้อมูล** — ข้อมูลในระบบที่ต้องการแสดง
- **ชื่อป้ายกำกับ** — ชื่อที่แสดงข้างหน้าค่า เช่น "ลูกค้า:" หรือ "ราคารวม:"
- **ประเภทการแสดงผล** — ข้อความ / ตัวเลข / สถานะ / วันที่ / ป้าย / ราคา

### 🏭 บล็อกเฉพาะสถานีงาน

บล็อกกลุ่มนี้ออกแบบมาสำหรับการทำงานในสถานีโดยเฉพาะ

#### เลือกลำดับสถานี (StationSequencePicker)

บล็อกสำหรับเลือกและเรียงลำดับสถานีที่ชิ้นงานต้องผ่าน เมื่อพนักงานสร้างออเดอร์ ระบบจะนำลำดับสถานีที่เลือกไว้ไปกำหนดเส้นทางการผลิตของชิ้นงานทุกชิ้น

**วิธีใช้งาน:**
1. บล็อกแสดงรายชื่อสถานีทั้งหมดที่มีในโรงงาน
2. กดเลือกสถานีที่ต้องการ
3. ลากเพื่อเรียงลำดับ
4. หรือเลือก Preset สำเร็จรูปเพื่อความรวดเร็ว

**Preset สำเร็จรูป:**

| Preset | เหมาะกับงาน |
|--------|-----------|
| กระจกนิรภัย | งานอบกระจกแบบ Tempered |
| กระจกลามิเนต | งานประกบฟิล์มกลาง |
| กระจกใส | งานตัดและขัดขอบธรรมดา |
| กระจกเจาะ | งานที่ต้องเจาะรูหรือขอบ |
| กระบวนการเต็ม | ผ่านทุกสถานี |

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ชื่อหัวข้อ | ข้อความที่แสดงด้านบนของบล็อก |
| ชื่ออ้างอิง | ชื่อที่ระบบใช้บันทึกลำดับสถานีที่เลือก เพื่อส่งให้ปุ่มสร้างออเดอร์นำไปใช้ |

---

#### ประวัติการผลิต (StationHistory)

แสดงประวัติชิ้นงานที่ผ่านสถานีนี้ รวมถึงชิ้นงานที่ทำเสร็จแล้วแต่ยังรอสแกนออก

**สิ่งที่แสดง:**
- ชิ้นงานที่ทำเสร็จแล้วแต่ยังไม่ได้สแกนออก (รอ Scan-out) — แสดงชัดเจนเพื่อให้พนักงานจัดการ
- ประวัติชิ้นงานที่ผ่านสถานีนี้ทั้งหมด จัดกลุ่มตามออเดอร์

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ชื่อหัวข้อ | ข้อความที่แสดงด้านบน |
| แสดงสูงสุด | จำนวนแถวประวัติ |

---

#### คิวงานปัจจุบัน (StationQueueBlock)

แสดงชิ้นงานที่กำลังอยู่ในสถานีนี้ จัดกลุ่มตามออเดอร์ พนักงานสามารถดำเนินการแต่ละชิ้นได้จากบล็อกนี้โดยตรง

**สิ่งที่แสดงและทำได้:**
- รายการชิ้นงานทั้งหมดที่อยู่ในสถานี จัดกลุ่มตามออเดอร์
- ปุ่ม **สแกนเข้า** — บันทึกว่าชิ้นงานมาถึงสถานีแล้ว
- ปุ่ม **เริ่มงาน** — บันทึกเวลาเริ่มทำงาน
- ปุ่ม **เสร็จสิ้น** — บันทึกว่าทำงานเสร็จแล้ว
- ปุ่ม QR Code — เปิด QR ของชิ้นงานเพื่อพิมพ์หรือส่งต่อ
- ข้อมูลอัปเดตอัตโนมัติตลอดเวลา
- รองรับการสแกนด้วยกล้อง

---

#### สต็อกวัสดุ (InventoryStockBlock)

แสดงสต็อกวัสดุในคลัง ณ ปัจจุบัน เหมาะสำหรับสถานีที่ต้องตรวจสอบว่าวัสดุพร้อมก่อนรับงาน

| ค่าตั้งค่า | ตัวเลือก |
|-----------|---------|
| ชื่อหัวข้อ | ข้อความแสดงด้านบน |
| แสดงสูงสุด | จำนวนรายการ |
| กรองตามสถานะ | ทั้งหมด / เฉพาะที่ใกล้หมด / เฉพาะที่หมดแล้ว |
| ช่องค้นหา | เปิด/ปิด |

---

#### แผงปล่อยออเดอร์ (OrderReleasePanel)

แผงสำหรับ Admin หรือ Manager ใช้ตรวจสอบและปล่อยออเดอร์เข้าสายการผลิต ระบบจะแสดงข้อมูลที่จำเป็นในการตัดสินใจ เช่น สต็อกวัสดุเพียงพอหรือไม่ ข้อมูลลูกค้าครบหรือไม่ ราคาถูกต้องหรือไม่

**สิ่งที่แสดงและทำได้:**
- รายการออเดอร์ที่รอปล่อยสู่การผลิต
- สัญลักษณ์แสดงว่าออเดอร์นั้น ✓ พร้อม หรือ ✗ มีปัญหา (สต็อก/ราคา/ข้อมูล)
- ปุ่มยืนยันปล่อยออเดอร์เข้าสายการผลิต
- ข้อมูลอัปเดตอัตโนมัติ

---

#### พิมพ์สติกเกอร์ (StickerPrintBlock)

ปุ่มสำหรับพิมพ์สติกเกอร์ QR Code ติดชิ้นงาน เมื่อกดจะแสดงรายการ Template สติกเกอร์ที่สร้างไว้ ให้เลือกแล้วพิมพ์

| ค่าตั้งค่า | รายละเอียด |
|-----------|-----------|
| ข้อความบนปุ่ม | เช่น "พิมพ์สติกเกอร์" |

**การทำงาน:**
1. พนักงานกดปุ่ม
2. ระบบเปิดหน้าต่างให้เลือก Template สติกเกอร์
3. ดึงข้อมูลชิ้นงานทุกชิ้นในออเดอร์/บิลที่เลือกอยู่
4. แสดงตัวอย่างสติกเกอร์พร้อมข้อมูลจริง
5. พนักงานกดพิมพ์

## ข้อมูลที่แสดงได้อัตโนมัติ

บล็อกที่รองรับ **แสดงข้อมูลอัตโนมัติ** สามารถเลือกข้อมูลต่อไปนี้ได้:

### จากออเดอร์ที่เลือกอยู่

| ชื่อข้อมูล | แสดงอะไร |
|-----------|---------|
| `order.code` | รหัสออเดอร์ |
| `order.customer.name` | ชื่อลูกค้า |
| `order.quantity` | จำนวนชิ้นงานทั้งหมด |
| `order.status` | สถานะออเดอร์ปัจจุบัน |
| `order.totalPrice` | ราคารวม |
| `order.dueDate` | วันกำหนดส่ง |

### จากบิลที่เลือกอยู่

| ชื่อข้อมูล | แสดงอะไร |
|-----------|---------|
| `request.code` | รหัสบิล |
| `request.customer.name` | ชื่อลูกค้า |
| `request.details.quantity` | จำนวนจากบิล |

### จากชิ้นงานที่สแกน

| ชื่อข้อมูล | แสดงอะไร |
|-----------|---------|
| `pane.glassType` | ประเภทกระจก |
| `pane.status` | สถานะชิ้นงาน |
| `pane.width` | ความกว้าง |
| `pane.height` | ความสูง |

### จากข้อมูลที่กรอกในฟอร์ม

| ชื่อข้อมูล | แสดงอะไร |
|-----------|---------|
| `form.ชื่ออ้างอิง` | ค่าที่พนักงานกรอกในช่องที่มีชื่ออ้างอิงนั้น |
